# 클러스터를 활용한 멀티 프로세스 WebSocket 서버 아키텍처

---

## 1. cluster.ts의 기능 설명

`cluster.ts`는 마스터 프로세스로서 클러스터 전체를 관리하고 로드 밸런싱을 담당하는 파일입니다. 주요 기능은 다음과 같습니다:

1. **워커 프로세스 관리**: 환경 변수로 지정된 수만큼 워커 프로세스를 생성하고 각 워커에 고유한 포트를 할당합니다.

2. **캔버스 기반 라우팅**: 캔버스 ID를 기반으로 특정 워커에 연결을 라우팅하여, 같은 캔버스의 모든 클라이언트가 동일한 워커에 연결되도록 보장합니다. 이를 통해 같은 캔버스를 사용하는 클라이언트들이 항상 같은 프로세스에서 데이터를 공유할 수 있어 데이터 일관성을 유지할 수 있습니다.

3. **로드 밸런싱**: 새로운 캔버스의 경우 연결 수가 가장 적은 워커를 선택하여 부하를 분산시킵니다. 각 워커의 연결 수를 실시간으로 추적하여 동적으로 부하를 분산합니다.

4. **워커 재시작 메커니즘**: 워커가 비정상 종료되면 자동으로 재시작하고, 관련 리소스(캔버스-워커 매핑, 연결 수 등)를 정리한 후 핸들러를 재등록합니다.

5. **프로세스 간 통신(IPC)**: 워커로부터 캔버스 등록, 연결 개방/종료 등의 메시지를 수신하여 클러스터 전체의 상태를 추적합니다.

6. **WebSocket 프록시**: 클라이언트의 WebSocket 업그레이드 요청을 적절한 워커 프로세스로 프록시하여 라우팅합니다.

---

## 2. server.ts의 기능 설명

`server.ts`는 워커 프로세스에서 실행되는 WebSocket 서버를 초기화하고 관리하는 파일입니다. 주요 기능은 다음과 같습니다:

1. **WebSocket 서버 초기화**: HTTP 서버 위에 WebSocket 서버를 생성하여 클라이언트 연결을 처리합니다.

2. **URL 기반 캔버스 식별**: 연결 URL에서 캔버스 ID를 추출하여 유효성을 검증하고, 잘못된 형식의 요청은 즉시 거부합니다.

3. **마스터 프로세스와의 통신**: IPC를 통해 마스터 프로세스에 캔버스 등록, 연결 개방/종료 이벤트를 전달하여 클러스터 전체의 상태를 동기화합니다. 이를 통해 마스터 프로세스가 각 워커의 상태를 실시간으로 파악할 수 있습니다.

4. **데이터베이스 연결 관리**: 서버 시작 시 MongoDB에 연결하고, 프로세스 종료 시 안전하게 연결을 해제합니다. 각 워커 프로세스가 독립적인 데이터베이스 연결을 가지도록 합니다.

## 3. 두 파일에서 자신 있는 이유

### 3.1 프로세스 단위 사고의 습득

이 프로젝트를 통해 **프로세스 단위로 시스템을 설계하고 생각할 수 있는 능력**을 기를 수 있었습니다. 단일 프로세스 애플리케이션과 달리, 멀티 프로세스 환경에서는 각 프로세스가 독립적인 메모리 공간을 가지며, 프로세스 간 통신(IPC)을 통해 상태를 공유해야 합니다.

- **프로세스 격리**: 각 워커 프로세스가 독립적으로 동작하여 하나의 워커에 문제가 발생해도 다른 워커에 영향을 주지 않습니다. 이는 단일 프로세스에서의 에러가 전체 시스템을 멈추게 하는 문제를 해결합니다.

- **프로세스 생명주기 관리**: 워커 프로세스의 생성, 모니터링, 재시작, 종료를 체계적으로 관리하는 방법을 학습했습니다. 특히 워커가 비정상 종료되었을 때 자동으로 재시작하고 리소스를 정리하는 메커니즘을 구현했습니다.

- **프로세스 간 상태 동기화**: 마스터 프로세스와 워커 프로세스 간의 상태를 IPC를 통해 동기화하는 방법을 구현했습니다. 각 워커의 연결 수, 캔버스 매핑 등의 정보를 실시간으로 추적하여 로드 밸런싱에 활용합니다.

### 3.2 확장 가능한 아키텍처 설계

- 클러스터 기반 멀티 프로세스 구조로 CPU 코어를 효율적으로 활용할 수 있습니다. Node.js의 단일 스레드 특성을 극복하여 멀티코어 시스템의 성능을 최대한 활용합니다.
- 워커 수를 환경 변수로 제어하여 운영 환경에 맞게 쉽게 조정 가능합니다. 서버의 하드웨어 사양에 따라 워커 수를 동적으로 변경할 수 있습니다.

### 3.3 스마트한 라우팅 전략

- 캔버스 ID 기반 스티키 세션을 통해 같은 캔버스의 클라이언트들이 항상 같은 워커에 연결되어 데이터 일관성을 보장합니다. 이는 실시간 협업 애플리케이션에서 매우 중요한 요구사항입니다.
- 동시에 연결 수 기반 로드 밸런싱으로 워커 간 부하를 균등하게 분산시킵니다. 새로운 캔버스는 항상 연결 수가 가장 적은 워커에 할당됩니다.

### 3.4 견고한 에러 처리 및 복구

- 워커 프로세스가 비정상 종료되어도 자동으로 재시작하여 서비스 가용성을 유지합니다. 프로세스 단위의 격리 덕분에 하나의 워커가 크래시되어도 다른 워커들은 정상적으로 동작합니다.
- 리소스 정리 로직을 통해 메모리 누수를 방지하고, 프록시 에러 시 소켓을 안전하게 종료합니다. 워커가 종료될 때 관련된 모든 리소스를 정리하여 시스템의 안정성을 보장합니다.

### 3.5 효율적인 상태 관리

- Map 자료구조를 활용하여 O(1) 시간 복잡도로 캔버스-워커 매핑과 연결 수를 관리합니다. 대량의 캔버스와 연결을 효율적으로 처리할 수 있습니다.
- 워커의 연결 수를 실시간으로 추적하여 동적 로드 밸런싱이 가능합니다. 각 워커의 부하를 지속적으로 모니터링하여 최적의 라우팅 결정을 내립니다.

### 3.6 명확한 책임 분리

- 워커 프로세스는 WebSocket 연결 처리에만 집중하고, 클러스터 관리는 마스터 프로세스에 위임하여 단일 책임 원칙을 준수합니다. 각 프로세스가 명확한 역할을 가지도록 설계했습니다.
- 각 워커가 독립적으로 동작하여 하나의 워커에 문제가 발생해도 다른 워커에 영향을 주지 않습니다.

## 4. cluster.ts 코드

<details>
<summary>코드 보기/숨기기</summary>

```typescript
import "dotenv/config";
import cluster, { Worker } from "cluster";
import http from "http";
import { createProxyServer } from "http-proxy";
import { startWorker } from "./server";

const PORT = Number(process.env.PORT);
const WORKER_COUNT = Number(process.env.WORKER_COUNT);

// 캔버스 ID → 워커 ID 매핑
const canvasToWorker = new Map<string, number>();
// 워커 ID → 연결 수 매핑
const workerConnections = new Map<number, number>();
// 워커 ID → Port 매핑
const workerPorts = new Map<number, number>();

if (cluster.isPrimary) {
  // 워커 리소스 정리 함수
  function cleanupWorkerResources(workerId: number) {
    for (const [canvasId, workerId] of canvasToWorker.entries()) {
      if (workerId === workerId) {
        canvasToWorker.delete(canvasId);
      }
    }
    workerConnections.delete(workerId);
    workerPorts.delete(workerId);
  }

  // 워커 이벤트 핸들러 설정 함수
  function setupWorkerHandlers(worker: Worker, workerId: number) {
    worker.on("message", (msg: any) => {
      if (msg.type === "canvas-registered") {
        canvasToWorker.set(msg.canvasId, msg.workerId);
      }
      if (msg.type === "connection-opened") {
        const count = workerConnections.get(msg.workerId) || 0;
        workerConnections.set(msg.workerId, count + 1);
      }
      if (msg.type === "connection-closed") {
        const count = workerConnections.get(msg.workerId) || 0;
        workerConnections.set(msg.workerId, Math.max(0, count - 1));
      }
    });

    worker.on("exit", () => {
      // 리소스 정리
      cleanupWorkerResources(workerId);

      // 워커 재시작
      const newWorkerPort = PORT + 1000 + workerId;
      const newWorker = cluster.fork({
        WORKER_ID: workerId,
        WORKER_PORT: newWorkerPort,
      });

      workerConnections.set(workerId, 0);
      workerPorts.set(workerId, newWorkerPort);

      // 핸들러 재등록
      setupWorkerHandlers(newWorker, workerId);
    });
  }

  // 워커 프로세스 생성
  for (let i = 0; i < WORKER_COUNT; i++) {
    const workerPort = PORT + 1000 + i;
    const worker = cluster.fork({
      WORKER_ID: i,
      WORKER_PORT: workerPort,
    });

    workerConnections.set(i, 0);
    workerPorts.set(i, workerPort);

    // 핸들러 등록
    setupWorkerHandlers(worker, i);
  }

  // HTTP 서버 생성
  const server = http.createServer();
  const proxy = createProxyServer({});

  // 프록시 에러 처리
  proxy.on("error", (err, req, socket) => {
    if (socket && !socket.destroyed) {
      socket.destroy();
    }
  });

  // WebSocket 업그레이드 요청 처리
  server.on("upgrade", (request, socket, head) => {
    const url = request.url || "";

    // URL 파싱
    const canvasMatch = url.match(/^\/canvas\/(.+)$/);
    if (!canvasMatch) {
      socket.destroy();
      return;
    }
    const canvasId = canvasMatch[1];

    // 캔버스 ID 기반으로 워커 선택
    let targetWorkerId: number;

    if (canvasToWorker.has(canvasId)) {
      // 매핑된 워커 선택
      targetWorkerId = canvasToWorker.get(canvasId)!;
    } else {
      // 연결 수가 가장 적은 워커 선택
      const sortedWorkers = Array.from(workerConnections.entries()).sort((a, b) => a[1] - b[1]);

      if (sortedWorkers.length === 0) {
        socket.destroy();
        return;
      }

      targetWorkerId = sortedWorkers[0][0];
    }

    // 선택된 워커의 포트로 프록시
    const targetPort = workerPorts.get(targetWorkerId);
    if (!targetPort) {
      socket.destroy();
      return;
    }

    proxy.ws(request, socket, head, {
      target: `http://localhost:${targetPort}`,
    });
  });

  server.listen(PORT);
} else {
  const workerId = Number(process.env.WORKER_ID);
  const workerPort = Number(process.env.WORKER_PORT);
  startWorker(workerId, workerPort);
}
```

## 5. server.ts 코드

<details>
<summary>코드 보기/숨기기</summary>

```typescript
import "dotenv/config";
import http from "http";
import { WebSocketServer } from "ws";
import { handleWebSocketConnection } from "./yjs/yjsHandler";
import { connectMongoDB, disconnectMongoDB } from "./mongodb/client";

export function startWorker(workerId: number, port: number) {
  const server = http.createServer();
  const wss = new WebSocketServer({ server });

  wss.on("connection", (ws, req) => {
    const url = req.url || "";

    // URL 파싱
    const canvasMatch = url.match(/^\/canvas\/(.+)$/);
    if (!canvasMatch) {
      ws.close(1008, "Invalid URL format");
      return;
    }
    const canvasId = canvasMatch[1];

    // 마스터 프로세스에 캔버스 등록 알림
    process.send?.({
      type: "canvas-registered",
      canvasId,
      workerId,
    });

    // 열림 알림
    process.send?.({
      type: "connection-opened",
      workerId,
    });

    handleWebSocketConnection(ws, req);

    // 연결 해제 알림
    ws.on("close", () => {
      process.send?.({
        type: "connection-closed",
        workerId,
      });
    });
  });

  // 서버 시작
  async function startServer() {
    try {
      await connectMongoDB();

      server.listen(port);

      server.on("error", () => {
        process.exit(1);
      });
    } catch (error) {
      process.exit(1);
    }
  }

  startServer();

  // 종료 처리
  process.on("SIGTERM", async () => {
    await disconnectMongoDB();
    server.close(() => process.exit(0));
  });

  // 인터럽트 처리
  process.on("SIGINT", async () => {
    await disconnectMongoDB();
    server.close(() => process.exit(0));
  });
}
```
